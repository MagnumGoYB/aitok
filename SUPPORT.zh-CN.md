# Support

[English](SUPPORT.md)

## 获取帮助

Bug report、功能请求和文档缺口请使用 GitHub issues。可用时请使用 issue templates。

## 创建 Issue 前

- 本地数据发现相关问题先运行 `aitok doctor`。
- 开发回归问题先运行 `make test`。
- 从日志中移除 API Key、prompt、私有路径和个人数据。
- 提供命令、时间范围、工具、预期结果、实际结果和操作系统。

## 支持范围

维护者可以协助 `aitok` 解析、CLI 输出、本地设置和 release artifacts。对于未通过本地 telemetry 记录的 Gemini CLI 历史用量，维护者无法恢复。
